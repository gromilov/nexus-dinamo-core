# 🧬 Проект «SPORE-NPM»: Архитектура Автономного Модуля

Этот документ описывает трансформацию текущей структуры `SYNDICATE-SPORE-CORE` в полноценный Node.js пакет для радикального упрощения развертывания новых ячеек.

## 🎯 Цели
1. **Унификация**: Замена Bash-скриптов на кроссплатформенный JS/TS.
2. **Скорость**: Инициализация через `npx` без клонирования всего репозитория.
3. **Чистота**: Манифест и Ядро лишены лишних «технических» файлов Базовой Споры.
4. **Обновляемость**: Легкое обновление логики протоколов через `npm update`.

---

## 🏗️ Структура Пакета (@syndicate/spore-core)

```
spore-core/
├── bin/
│   └── cli.js           # Точка входа для 'npx spore'
├── src/
│   ├── core/            # Логика протоколов (Resonance, Security, Dormancy)
│   ├── templates/       # Шаблоны (Brains, Astro-site)
│   └── utils/           # Диагностика, Транслитерация
├── package.json
└── README.md
```

---

## ⌨️ Интерфейс CLI (Command Line)

### 1. `npx spore init`
- Запускает интерактивный опрос (Inquirer.js).
- Генерирует файловую структуру в текущей папке:
  - `.СИНДИКАТ_ЯДРО.md`
  - `brains/`
  - `manifest-site/` (на базе Astro)
  - `protocols/` (импортируются из модуля)

### 2. `npx spore add-brain`
- Добавляет новую сущность в существующее ядро.
- Автоматически обновляет реестр в `.СИНДИКАТ_ЯДРО.md`.

### 3. `npx spore doctor`
- Проверка резонанса (сеть, версии Node.js, лимиты).

## 🧬 Технологический Стек
- **Runtime**: Node.js 22+ (текущая норма Astro).
- **Core**: TypeScript (для строгой типизации личностей).
- **CLI Helpers**: `commander`, `inquirer`, `picocolors`.
- **Template Engine**: `handlebars` (для гибкой подстановки данных в PERSONA.md и Astro).
